Understanding Linux Processes

A process is essentially an executing instance of a program. In Linux, the operating system kernel manages all hardware and software interactions by spawning processes and assigning them resources (CPU time, Memory).

Every process in Linux has a unique identifier called a PID (Process ID). When the Linux system boots, it starts process PID 1 (usually systemd or init), which is the ancestor of all other processes.

Types of Processes

  • Foreground Processes: Run interactively on the terminal. The terminal is blocked until the process finishes.
  • Background Processes: Run non-interactively. You append an ampersand & to the end of a command to run it in the background, freeing your terminal.
  • Daemons: Special background processes that start at boot and run continuously, listening for requests (e.g., sshd for SSH, httpd for web traffic).
  • Zombies: Processes that have completed execution but still have an entry in the process table because their parent hasn't read their exit status.

Common Commands & Usage

Knowing how to enumerate and kill processes is a vital skill for both system administrators and hackers.

BASH 
# List processes running in the current terminal session
$ ps

# View a rich snapshot of ALL processes running on the system
$ ps aux

# View processes in a dynamic, continuously updating dashboard (like Task Manager)
$ top
$ htop       # (If installed, much better visualization than top)

# Find the PID of a specific process using grep
$ ps aux | grep "apache2"
$ pgrep apache2

# Terminate a process gracefully using its PID
$ kill 1234

# Force-kill a process aggressively (SIGKILL)
$ kill -9 1234
$ killall apache2

Security Implications

During a penetration test or incident response scenario, monitoring processes is crucial:

  • Rogue Daemons: Attackers will often spawn reverse shells or backdoors hidden as innocent-looking processes (e.g., naming malware `bash` or `kworker`).
  • Resource Exhaustion: Malware like cryptominers will spike CPU usage, which is immediately visible in top.
  • Elevated Execution: Looking for processes running as root that shouldn't be can indicate privilege escalation vulnerabilities.
