Package Management Basics

A Package Manager automates the process of installing, upgrading, configuring, and removing computer programs for a computer's operating system in a consistent manner. Instead of manually downloading an executable and figuring out its dependencies, you use a package manager to fetch it securely from centralized repositories.

Different Linux distributions use different package managers and package formats.

The Package Manager Families

Debian Base (APT / dpkg)

Used by Ubuntu, Kali Linux, and Debian.

  • Packages: .deb
  • Frontend: apt, apt-get
  • Backend: dpkg

Red Hat Base (YUM / DNF / RPM)

Used by RHEL, Fedora, and CentOS.

  • Packages: .rpm
  • Frontend: dnf, yum
  • Backend: rpm

Arch Base (Pacman)

Used by Arch Linux and Manjaro.

  • Packages: .pkg.tar.zst
  • Frontend/Backend: pacman

Common Commands (APT Focus)

Because Kali Linux and Ubuntu are the most common platforms for cybersecurity, learning APT is essential.

BASH 
# 1. Update the local index of available packages (Do this first)
$ sudo apt update

# 2. Upgrade all software on the system to newest versions
$ sudo apt upgrade

# 3. Search for a specific tool in the repository
$ apt search nmap

# 4. Install a new tool securely
$ sudo apt install nmap

# 5. Remove a tool completely
$ sudo apt remove nmap

# 6. Install a raw local .deb file (Bypasses APT, uses DPKG)
$ sudo dpkg -i software_pack.deb

Security Considerations

  • Supply Chain Attacks: Package managers verify software using cryptographic signatures (GPG keys) to ensure it hasn't been backdoored in transit. Be extremely careful when adding unknown 3rd party repositories (PPAs).
  • Privilege Escalation: If a user has `sudo` rights specifically to run `apt` or `dpkg`, an attacker can easily escalate to root by building a malicious `.deb` package or abusing `apt` hook scripts. Be wary of giving users `sudo apt` without a password.
